What is Ethical Hacking

Ethical hacking involves an authorized attempt to gain unauthorized access to a computer system, application or data. Carrying out an ethical hack involves duplicating techniques and actions of malicious hackers. This practice helps to identify security vulnerabelities which can then be resolved before a malicious attacker hackers the opportunity to exploit them.


Ethical hackers also known as "White Hats" are security experts that perform these assessments. The proactive work they do helps to improve an organization's security posture or situation.

Are Ethical Hackers Different From Crackers

Malware, short for malicious software, is a blanket term for viruses, worms, trojan and other harmful computer programs hackers use to wreak destruction and gain access to sensitive information. This act is also known as "Black hats" or "Cracking".
Ethical hackers are rather interested in finding weaknesses in your security systems and helping or giving the organization measures or techniques to make the system stronger again and prevent malicious hackers or crackers from gaining access to the security system.

Steps Involved In Ethical Hacking

  • Reconnaisance

  • Scanning

  • Exploitation

  • Maintaining Access

  • Covering Tracks

  • Reporting

STEP 1: RECONNAISANCE


Reconnaisance is a set of processes or techniques(Footprinting, Scanning and Enumeration) used to covertly discover and collect information about a target system as possible. Such information includes:

  • IP Address
  • IP Address Range
  • Network
  • DNS Records(Domain Name Server)

    • Some tools used for Reconnaisance include:

nslookup

whois lookup

nmap

STEP 2: SCANNING

Scanning is a set of procedure for identifying Live hosts,ports and services,discovering Operating Systems and architecture of target systems,identifying vulnerabelities and threats in the network.
Scanning refers to collecting more information using complex and aggressive Reconnaisance techniques. Some areas which are scanned for information includes:

  • Active Ports
  • Active Hosts
  • Services
  • Application and OS

    • Some tools used for Scanning processes are:

Openvas

Nessus

Wireshark

STEP 3:EXPLOITATION

Exploitation is a piece of programmed software or script which can allow hackers to take control over a system,exploiting its vulnerabelities. Hackers normally use vulnerabelity scanners like Nessus,Nexpose,Beef,Metasploit and SQL map to find these vulnerabelity. In Exploitation, the hacker selects the right attack,launches the attack and gains access to the computer system.

Nexpose

Beef

SQL map

STEP 4: MAINTAINING ACCESS

Maintaining Access is the process where the hacker has already gained access to a system. After gaining access, the hacker installs some backdoors in order to enter into the system when he needs access in this owned system in future.Metasploit is the preferred tool in this process by most hackers. Other tools include Powersploit and Weebly. Maintaining access can also be done by creating new users or escalating privilages.

Metasploit

Powershell

Weebly

STEP 5 : COVERING TRACKS

Covering is the process of concealling one's whereabouts,activities,intentions,or the evidence of one's involvement. The above clearly shows that after hackers gain access to your computer system and maintain its access, they then cover or clean their tracks by clearing cache and cookies,tampering log files and closing of ports,stopping of services and creating firewalls to block external users. One way of covering your tracks over a network is using reverse HTTP shells. In this type of attack, the attacker installs a reverse HTTP shell on the victim's computer. This reverse HTTP shell is coded in such a way that it ask commands from an external moderator who controls this reverse shell at regular intervals. So it looks like a normal traffic to an organization network perimeter security device like a firewall, as port 80 is usually opened.

STEP 6: REPORTING

This stage of hacking is very important; it differentiate one from a hacker or cracker. Here one informs the right organization about a weakness in security systems and gives them advice on how to improve on it to prevent a craker from breaking into their system.
